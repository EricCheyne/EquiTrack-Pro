# Architecture Decision Records

Architecture Decision Records (ADRs) document the significant architectural decisions made during the development of EquiTrack Pro.

## Format

Each ADR follows the [MADR](https://adr.github.io/madr/) format:

- **Title**: One sentence summary
- **Status**: Proposed, Accepted, Deprecated, Superseded
- **Context**: The issue or problem
- **Decision**: What decision was made
- **Consequences**: Implications of the decision
