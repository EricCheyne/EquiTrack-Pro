# ADR-002: Use Next.js for Frontend

**Status:** Accepted

## Context

We need a modern, performant frontend framework that:
- Provides server-side rendering capabilities
- Has strong tooling and ecosystem
- Supports TypeScript
- Offers good developer experience

## Decision

We chose Next.js as our frontend framework.

## Consequences

**Positive:**
- Built-in SSR and static generation
- Excellent performance optimization
- Strong TypeScript support
- Great developer experience with hot reload
- Comprehensive documentation

**Negative:**
- Opinionated framework structure
- Learning curve for new developers
- Can be overkill for simple applications
